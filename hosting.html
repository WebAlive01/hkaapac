PMWeb Hosting & SaaS — Flexible Deployment for APAC Organisations

PMWeb can be acquired as Software as a Service (SaaS) — with all hosting included in the fee — or purchased and hosted on your own systems, by PMWeb, or in any data centre or cloud facility. Our primary recommendation for most organisations is SaaS via Amazon Web Services, with Australian data residency available.

Three Ways to Run PMWeb

[NEW — VERIFY] Section heading written from audit brief. Body below is verbatim from existing site.

PMWeb is a 3-tier Windows application with a browser-based front-end, IIS-based ASP.Net Business Logic Layer and MS SQL-Server backend. PMWeb can be hosted in Physical-only, Virtual-only or a mix of physical and virtual server infrastructure. If PMWeb is acquired as Software as a Service (SaaS), all hosting is included in the fee.

Primary Recommendation

Software as a Service (SaaS)

PMWeb Software as a Service (SaaS) is a popular means to access PMWeb's powerful features without purchasing the Modules and licences. SaaS can be a cost-effective alternative for managing individual projects or a group of projects for up to three or four years.

All hosting included in the SaaS fee
Delivered via Amazon Web Services (AWS) — all data restricted to the Australian availability zones
Operational in its out-of-the-box form within seven (7) business days of signing the SaaS Agreement
Minimum term 12 months; renewable annually thereafter
1TB storage included (scalable for larger implementations)

SaaS on AWS — Our Primary Recommendation

[NEW — VERIFY] Overline text written from audit brief (client directive: SaaS leads). All body copy below is verbatim from the existing /pmweb-in-detail-saas page.

What's Included in PMWeb SaaS

The standard SaaS package comprises: PMWeb's Core Module with Business Intelligence Reporting; Project Management (including Contract Management and Cost Management); Document Management; Visual Workflow; Asset Explorer; the required number of PMWeb Licences; Hosting by PMWeb; 1TB storage (higher for larger numbers of Licences); and Unlimited Internet.

SaaS is a popular alternative where cashflow is a consideration, as there is no purchase involved. It can also be used to evaluate PMWeb software through use in the field, prior to proceeding with purchase. Typically, by Year 3 or Year 4, it is more cost-effective to purchase the software and licences compared to the ongoing costs of SaaS — however, some Clients prefer SaaS because it removes the responsibility for maintaining an IT infrastructure.

PMWeb SaaS is operational in its out-of-the-box form within seven (7) business days of signing the SaaS Agreement. Implementation can be fast-tracked by running Training and initial configuration in parallel to suit Client requirements — see details on our Quick Start Packages which can complete this training and set-up in 10–15 days according to specific Client requirements.

Facility Locations

SaaS Available from Multiple Locations

PMWeb SaaS is available from multiple locations. For APAC organisations, the Australian availability zone via AWS is the recommended default, ensuring all data is retained in Australia.

Australia (Recommended)

Delivered via Amazon Web Services (AWS) and the Amazon EC2 Cloud. All data is restricted to the Australian availability zones, ensuring all data is retained in Australia. AWS has been the world's #1 provider of Cloud infrastructure for ten years in a row (Gartner Magic Quadrant, August 2020).

USA

Hosted by PMWeb, typically using the AWS EC2 Cloud. This SaaS solution presents some economies for organisations that do not require data to be retained solely in Australia.

Other Global Locations

Ireland, Germany, England, Singapore, Hong Kong, Japan, Korea, India, Brazil and Central Canada. Client may choose the location that best suits their data residency and latency requirements.

On-Premise & Hosted Requirements

Installation Requirements for PMWeb

For organisations hosting PMWeb on their own systems or through a third-party data centre, typical installation requirements are set out opposite. PMWeb is a true zero-footprint web-based solution. Browsers compatible with PMWeb include Microsoft Edge, Google Chrome, Firefox and the latest versions of Internet Explorer 11, operating in Microsoft Windows 7 and Windows 10 environments.

The installation options are: Physical-only infrastructure; Physical/Virtual Hybrid infrastructure; or Virtual-Only infrastructure. In each case, the actual number of Users that can be served by the system depends on the number of CPUs, CPU speed, memory, and other factors.

Physical-Only Infrastructure

The web-code is installed on dual quad-core CPUs with RAID storage, load balanced using either hardware load-balancers or the Network-Load-Balancing capabilities of the Windows OS. The MSSQL back-end is run on dual quad-core CPUs with RAID or shared storage using SQL-Based clustering for high-availability and load-balancing.

Physical/Virtual Hybrid Infrastructure

The web-code is installed in Virtual Machines running on dual quad-core CPUs with RAID storage, load balanced using either hardware load-balancers or the VMWare/Microsoft Virtual infrastructure load balancing/failover capabilities (i.e.: vMotion). The MSSQL back-end is run on dual quad-core CPUs with RAID or shared storage using SQL-Based clustering for high-availability and load-balancing.

Virtual-Only Infrastructure

The web-code is installed in Virtual Machines running on dual quad-core CPUs with RAID storage, load balanced using either hardware load-balancers or the VMWare/Microsoft Virtual infrastructure load balancing/failover capabilities (i.e.: vMotion). A SAN is recommended for this scenario; the interface can be iSCSI, Fibre or FCOE. The MSSQL back-end is run in Virtual Machines with SQL-Based clustering for high-availability and load-balancing.

Typical Installation Requirements

External URL
An external Web URL is required to licence PMWeb. IP addresses will not be licensed.
Operating System
Microsoft Windows Server 2008 R2 or Windows Server 2012 — check for latest approved versions for use with PMWeb.
Database Software
Microsoft SQL 2008 R2, 2012, 2014, or 2016 — check for latest approved versions. Must use default collation: SQL_Latin1_General_CP1_CI_AS.
Reporting Software
Microsoft SQL Report Server (included with SQL).
Visual Studio
Must be available on the network for installations where all data is to be retained at the client site — check for approved versions.
IIS
Enable .Net 4.5. Enable IIS Management Console. Enable All Application Development Features (.NET Extensibility, ASP, ASP.NET, etc.).
Internet Connection
DSL/Cable, Fibre Optic, or T1/T3.
